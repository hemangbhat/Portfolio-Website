<summary><b>🟣 OMERO — Omni-Modal Enterprise Research Orchestrator</b></summary>

<br/>

A multi-tenant enterprise research platform that ingests PDF and audio documents, stores them as vector embeddings, and answers research queries through an internal retrieval pipeline with optional, redacted external AI delegation.

| Aspect | Detail |
| :--- | :--- |
| **Stack** | Next.js 16 · React 19 · TypeScript · Python 3.11 · PostgreSQL · pgvector · Drizzle ORM |
| **Scale** | Multi-tenant SaaS layer — orgs, workspaces, usage metering, plan gating |
| **Performance** | Async ingestion queue, connection pooling, query caching, batch embeddings |
| **Security** | JWT auth, RBAC, document-level access control, PII redaction before delegation |
| **Impact** | 12-phase build · 306+ tests including 34+ Hypothesis property-based tests |
| **Repository** | [github.com/hemangbhat](https://github.com/hemangbhat) |

Built around a deterministic agent graph that treats the LLM as one untrusted node — validation, retrieval, and fallback live in tested code. A dedicated redaction layer guarantees private document content never crosses the tenant boundary, raising a `ContentLeakError` if verbatim chunks are detected in outbound payloads.

</details>
